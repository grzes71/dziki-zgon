# World Studio Specification (Spec-Driven Development)

**Version:** 2.1

## Purpose

World Studio is a desktop GUI editor for designing the game world of an Atari 8-bit adventure game.

It edits only YAML world data.

**YAML is the Single Source of Truth.**

ASM generation is performed later by **World Builder**.

---

- PySide6 (Qt)

# Project Structure

```text
world/

    world.yaml
    colors.yaml
    objects.yaml

    WHITE_FIELD/
        region.yaml
        screens/
            000.yaml
            001.yaml
            ...
```

The application opens the root **world/** directory, loading only YAML data.
`game.fnt` is not part of this directory and must be loaded separately via the GUI for visualization purposes.

---

# Input Files

## world.yaml

Global project configuration.

Reserved for future project-wide settings.

## colors.yaml

Defines the ANTIC 4/5 palette.

```yaml
BACKGROUND: [0,0,0]
PF0: [132,59,0]
PF1: [193,120,29]
PF2: [3,80,0]
PF3_INV: [0,48,139]
```

Changing the palette shall immediately refresh every preview.

## game.fnt

Raw Atari charset. Loaded separately via a specific GUI action (e.g., `File -> Load Charset...`) from outside the `world/` directory (e.g., the `fonts/` directory).

Requirements:

- 1024 bytes
- 128 characters
- ANTIC 4/5 encoding

World Studio SHALL render the charset exactly as Atari would.

## objects.yaml

Generated by Object Studio.

Contains:

- ObjectDefinition
- ObjectSize
- Flags
- Tile layout

Loaded in read-only mode.

## region.yaml

```yaml
id: WHITE_FIELD
name: White Field

layout:
  rows: 20
  columns: 20

start_screen: TAVERN
music: WHITE_FIELD
```

Supported operations:

- Create Region
- Rename Region
- Delete Region
- Resize Region
- Change Start Screen
- Change Music

Resizing automatically creates or removes screen YAML files.

## Screen YAML

```yaml
id: TAVERN

exits:
  north: null
  south: VILLAGE
  west: null
  east: CROSSROADS

objects:

  - object: HOUSE_SMALL
    x: 1
    y: 5

  - object: FENCE_1
    x: 0
    y: 0
    repeat-x: 40
```

# Rendering Engine

The rendering engine combines:

- game.fnt
- colors.yaml
- objects.yaml

to render the screen exactly as it will appear in-game.

Changing any of these files refreshes previews automatically.

---

# Region View (Mode 1)

Displays the logical region grid.

Each cell represents one Screen.

Double-click opens the Screen Editor.

---

# Live Region View (Mode 2)

World Studio SHALL provide an alternative **Live Region View**.

Instead of numbered cells, every Screen is rendered as a scaled thumbnail using:

- game.fnt
- colors.yaml
- objects.yaml

The entire Region is visible at once.

Example:

```text
+------+------+------+------+
|thumb |thumb |thumb |thumb |
+------+------+------+------+
|thumb |thumb |thumb |thumb |
+------+------+------+------+
```

The user shall be able to:

- Zoom in/out
- Pan
- Select a Screen
- Double-click to edit
- Toggle screen identifiers
- Toggle exit arrows
- Highlight unreachable Screens
- Refresh automatically after editing

This mode is intended to verify visual continuity between neighbouring Screens.

---

# Screen Editor

Canvas uses Atari character coordinates.
Default screen dimensions are **40x10** characters (ANTIC 5 gameplay area), with options in the GUI to adjust these dimensions if needed.

Objects snap to the character grid.

Object origin is always the **top-left corner**.

No anchor point exists.

Supported operations:

- Place Object
- Move Object
- Delete Object
- Duplicate
- Copy
- Paste
- Undo
- Redo
- **Set Player Start** (Updates `start_position` and `start_screen` in `world.yaml`, visually marking it on the canvas)

---

# Object Palette

Object thumbnails are generated dynamically from:

- game.fnt
- colors.yaml
- objects.yaml

---

# Repeat Placement

Support:

- repeat-x
- repeat-y

Preview repeated objects exactly as exported.
**Constraint**: The preview must accurately clip repeated objects at the edges of the Screen Canvas (e.g., stopping at X=39 or Y=9), mimicking the World Builder parser behavior.

---

# Exit Editor

Each Screen defines:

- north
- south
- east
- west

Only logical Screen IDs are shown.

Numeric IDs remain an internal World Builder detail.

---

# Validation

Errors:

- duplicate Region IDs
- duplicate Screen IDs
- invalid Exit destination
- unknown object
- object outside screen
- invalid repeat values

Warnings:

- unreachable Screens
- isolated Screen groups
- overlapping blocking objects

Saving fails if any error exists.

---

# Output

World Studio edits only:

- world.yaml
- colors.yaml
- region.yaml
- screen YAML files

It never generates ASM.

# GUI Layout

To save vertical space and allow large canvas rendering, the central area uses a **Tabbed or Switchable View**:

```text
+------------------------------------------------------------------+
|             | [Tab: Live Region] [Tab: Screen Canvas]            |
| Region Tree |                                                    |
| Object Pal. | (Double-clicking a screen in Live Region switches  |
|             |  to the Screen Canvas tab)                         |
+------------------------------------------------------------------+
| Properties | Preview | Status Bar                                |
+------------------------------------------------------------------+
```

# Acceptance Criteria

- Opens a world project (folder containing YAML files).
- Supports loading `game.fnt` as a separate, distinct operation.
- Loads colors.yaml and objects.yaml automatically from the world directory.
- Renders Atari screens accurately.
- Supports Region Grid and Live Region View.
- Displays generated object thumbnails.
- Supports graphical editing of Screens.
- Supports repeat-x / repeat-y.
- Saves deterministic YAML.
- Passes Ruff, Black, mypy and pytest.
